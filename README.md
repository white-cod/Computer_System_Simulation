# Computer_System_Simulation

### **1. Структура файлів**

---

- `hardware/` - директорія для всіх підсистем:
  - `__init__.py` - файл для ініціалізації пакета.
  - `cooling.py` - система охолодження.
  - `io_systems.py` - системи введення (миша, клавіатура) та виведення (монітор, динаміки).
  - `motherboard.py` - система зв'язку та маршрутизації.
  - `power.py` - система енергозабезпечення.
  - `processing.py` - процесорна та графічна підсистеми.
  - `storage.py` - система зберігання даних (ОЗП, ПЗП, BIOS).
- `actors.py` - файл з описом користувача.
- `core.py` - головний клас системи "Комп'ютер".
- `main.py` - головний файл для запуску симуляції.

---

### **2. Опис сутностей**

---

**Файл:** `actors.py`
Клас `User` (Користувач), який взаємодіятиме з системою.

- **Атрибути:** ім'я.
- **Методи:**
  - `press_power_button()`: ініціює запуск системи.
  - `input_command()`: передає текстові дані (через систему введення).
  - `move_mouse()`: передає координати графічному інтерфейсу.

---

### **3. Hardware**

**Файл:** `power.py`

- **Клас `PowerSupply` (Блок живлення):**
  - _Атрибути:_ потужність (за вимогами - не менше 500 Вт), вхідна напруга (220-240 В), поточний стан роботи (увімкнено/вимкнено).
  - _Методи:_ `supply_power()` (подає стабільне живлення на компоненти), `cut_power()` (знеструмлення).

**Файл:** `cooling.py`

- **Клас `CoolingSystem` (Система охолодження):**
  - _Атрибути:_ цільова температура (не вище 35°C), рівень шуму (не більше 40 дБ).
  - _Методи (за State Machine diagram охолодження):_ `passive_cooling()` (вентилятори на мінімалках), `active_cooling()` (вентилятори на максимумі при високому навантаженні), `stop_cooling()`.

**Файл:** `storage.py`
Тут створюємо базовий клас пам'яті та успадковуємо від нього конкретні типи.

- **Клас `BIOS`:**
  - _Методи:_ `run_diagnostics()` (тестує обладнання), `load_bootloader()` (передає керування системі для завантаження ОС).
- **Клас `RAM` (Оперативна пам'ять):**
  - _Атрибути:_ об'єм (не менше 16 ГБ), наявність збережених даних (тимчасовий кеш).
  - _Методи:_ `load_data()`, `clear_memory()` (згідно з діаграмою очищення перед вимкненням).
- **Клас `PersistentStorage` (HDD/SSD):**
  - _Атрибути:_ тип (наприклад, SSD), об'єм (від 512 ГБ), швидкість читання (від 3000 МБ/с).
  - _Методи:_ `read_os_files()`, `save_user_data()`.

**Файл:** `processing.py`

- **Клас `CPU` (Центральний процесор):**
  - _Атрибути:_ кількість ядер (не менше 6), частота (від 3.0 ГГц), стан (очікування команд / обробка даних).
  - _Методи:_ `execute_instructions()` (виконання логічних операцій), `request_data_from_ram()`.
- **Клас `GPU` (Графічний процесор):**
  - _Методи:_ `render_graphics()` (завантаження текстур та генерація кадрів).

**Файл:** `io_systems.py`

- **Клас `InputSystem` (Клавіатура, Миша):**
  - _Атрибути:_ наявність укр. розкладки, роздільна здатність миші (від 1000 DPI).
  - _Методи:_ `register_action()` (перетворення натискання/руху у цифровий сигнал).
- **Клас `OutputSystem` (Монітор, Динаміки):**
  - _Атрибути:_ роздільна здатність (1920x1080+), частота оновлення (від 60 Гц), стан (режим готовності / енергозбереження).
  - _Методи:_ `display_image()`, `play_sound()`.

**Файл:** `motherboard.py`

- **Клас `Motherboard`:**
  - _Атрибути:_ мережеві інтерфейси (Wi-Fi 6, Bluetooth 5.0).
  - _Методи:_ `synchronize_buses()` (синхронізація шин при увімкненні), `route_data()` (маршрутизація даних між внутрішніми компонентами).

### **4: Інтеграція у систему**

**Файл:** `core.py`

- **Клас `Computer`:**
  - _Конструктор (Ініціалізація):_ Приймає екземпляри блоку живлення, материнської плати, CPU, GPU, ОЗП, ПЗП, охолодження та периферії.
  - _Атрибут стану:_ Зберігає поточний стан системи (`State`: OFF, Booting, Ready, Working, Sleep)
  - _Метод `turn_on()` (Послідовність увімкнення за Sequence diagram):_
    1. Зміна стану на "Booting".
    2. Блок живлення подає струм.
    3. Материнська плата синхронізує шини.
    4. BIOS проводить діагностику.
    5. Зчитування ОС з SSD в RAM.
    6. Передача керування CPU.
    7. Зміна стану на "Ready to work".
  - _Метод `process_user_task()` (Обробка даних за Activity/Sequence diagram):_
    1. Зміна стану на "Working / Processing".
    2. Прийом даних від системи введення.
    3. Маршрутизація команди через материнську плату до CPU.
    4. Процесор обчислює дані (за потреби взаємодіючи з RAM).
    5. GPU генерує кадри - Система виведення відображає результат.
    6. Виклик системи охолодження (перевірка температури та запуск вентиляторів, якщо високе навантаження).
    7. Повернення стану в "Ready".
  - _Метод `save_work()` (Збереження результатів):_
    1. Читання фінальних даних з кешу/RAM.
    2. Запис блоків у пам'ять SSD.
  - _Метод `turn_off()` (Вимкнення комп'ютера):_
    1. Зупинка всіх програм.
    2. Вивантаження даних (очищення ОЗП).
    3. Сигнал на материнську плату та блок живлення.
    4. Відключення струму та зміна стану на "OFF".
